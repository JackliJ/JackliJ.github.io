Hexo3.0 的安装和本地调试(默认你已经安装好Git软件了)

网上比较出名的几篇博客年代都比较久远了，原先照着16年大神的博客去做的环境搭建，死的好惨

总结安装步骤：

1. 安装node.js的版本建议是最新的，后面安装npm的一些语法才不会不支持 附上官网链接
node.js
2. 其次就是hexo的安装 安装hexo 请严格执行以下命令
使用npm安装hexo：npm install hexo-cli -g
如需卸载hexo：npm uninstall hexo -g
查看hexo版本：hexo -v，验证是否安装成功。
3.hexo的初始化 在blog文件下面 你们也可以新建一个 文件夹中右键git bash 在git bash 执行命令 如果你配置了git 环境变量 也可以按住Shift在本文件夹打开命令窗口进行执行
*.新建一个blog文件夹，cd到blog文件夹下，打开Git Bash。
*.hexo初始化：hexo init
*.安装依赖包：npm install
*.初始化完成，在blog下就会生成以下文件目录：

4.hexo本地生成静态页面

hexo clean # 清理本地静态文件；
hexo generate # 生成静态页面，即public文件夹；
hexo server # 启用hexo本地服务器；
注：Hexo 3.0 把服务器独立成了个别模块，您必须先安装 hexo-server 才能使用。
hexo-server安装命令：npm install hexo-server –save

踩坑 执行hexo命令一定要在你初始化的文件夹下 执行hexo server命令 请一定要先安装hexo-server模块 hexo.3.0的启动服务模块是单独独立出来的 hexo初始化的文件夹请对比上文 保证不多不少

5.完成上面步骤 打开浏览器在地址栏输入http://localhost:4000即可本地访问静态博客页面。

GitHub的申请注册和项目建立

1. GitHub就安装官网的方式进行注册就好了 附上链接 GitHub
2.注册完成后 点击头像旁边的+号按钮 选择 New repository 新建一个项目
3.这里请注意 建立项目的名称 必须和你GitHub的昵称一致 比如我的Github名称是 jacklij 所以 我的名称就是jacklij.github.io github.io的后缀也是必须的 这样Github才会为你默认开启github pages 你才能使用博客
4. 所以 你新建项目的名称必然是 你的GitHub昵称.github.io 请确认这一点
5. 剩下的不用管了 然后底下按钮直接建立 进入setting里面 找到github pages 上面的链接 就是你后面的博客链接 一般也就是上面注册的名称
6. GitHub强制使用Https进行链接 所以你必须配置SSH公钥到github上 生成SSH秘钥 请百度Google
7. 将id_rsa.pub 文件内容全部复制 打开方式可以选择文本打开 在你的github网页点击头像 最下面 设置(setting) 进去之后 选择SSH keys 新建一个 New SSH Key 将刚刚复制的公钥粘贴进去 保存
8. 本地测试SSH是否配置成功 cmd输入ssh -T git@github.com 进行验证 输入yes 按回车

将Hexo静态网页提交到Github上

**1.**修改站点配置文件_config.yml：
deploy:
type: git
repo: git@github.com:你的GitHub昵称/你的GitHub昵称.github.io.git
branch: master
**2.**安装Git部署插件：npm install hexo-deployer-git –save

**3.**部署：

hexo clean # 清理本地静态文件；
hexo generate # 生成静态页面，即public文件夹；
hexo deploy # 部署到github
**4.**部署完毕，blog下的文件目录如下：

站点搭建完毕，打开浏览器在地址栏输入以下链接可随时访问自己的博客了。

github pages：http://yourname.github.io

如果你本地调试没有问题 远端没有变化或乱码 请尝试 hexo clean -> hexo g-> hexo d

将在后面介绍 Hexo主题Next的配置和使用

至此 你的博客就算搭建完毕了 如果你有任何疑问 欢迎联系www.lijin@foxmail.com

Windows 下openssl安装与配置

网上方法很多，大部分是针对32位机的,自己的电脑因为是win7,64位，摸索了很久才安装成功.

环境
WIN7, 64位， vs2005

下载ActivePerl
配置过程中需要生成一些mak文件，这些生成代码用perl脚本生成，所以要安装一个ActivePerl.

网址: http://www.activestate.com/activeperl/

我下载了两个版本:

[css] view plain copy 在CODE上查看代码片派生到我的代码片
ActivePerl-5.16.2.1602-MSWin32-x64-296513.msi
ActivePerl-5.16.2.1602-MSWin32-x86-296513.msi

装的是第一个版本，第二个版本没装成功.

下载openssl
网址 http://www.openssl.org/
我用的是openssl-0.9.8g版本, 没试过其它版本是否可行.
把openssl-0.9.8g.tar.gz解压到c盘根目录.

安装步骤：
(可以参照openssl目录下的install.win32)
1打开命令行
进入openssl源码目录。
cd c:/openssl-0.9.8.g
执行perl Configure VC-WIN32

2执行运行ms\do_ms
另外两种方式如果使用也必须保证本机有编译器才能使用。
:ms\do_masm（默认vc自带的编译器；也也以自己下载安装）
:ms\do_nasm（需要自己下载）

3 DOS窗口转到C:\Program Files (x86)\Microsoft Visual Studio 8\VC\bin目录
执行vcvars32.bat以配置环境变量.

4、跳到openssl目录下，编译动态链接库
cd c:/openssl-0.9.8.g
执行 >nmake -f ms\ntdll.mak

如果编译成功，输出在out32dll目录下：包括可执行文件、两个dll和两个lib文件: libeay32.dll, libeay32.lib, ssleay32.dll,ssleay32.lib

这里要特别强调一下，这种方法生成的是动库链接库, 也就是我们的应用工程如果要使用openssl, 编译时要指定
lib文件，程序发布时要用到相应的dll. 我遇到过一种情况是, 在win7下发布的使用openssl的程序无法运行，弹出
“运行时初始化相关的错误”, 但是在xp下编译，发布，在其它win7和winXP上都能运行. 估计是win7下没配置好，导致只能本机编译，
运行，无法发布.

如果要把openssl编译成静态的库，只要把上面的ntdll.mak换成nt.mak就可以了，我试着用了一个静态的库，
发现要解决很多编译时的兼容，冲突问题，建议不要使用静态方法.

(注意虽然是64位win7, 下生成的也是out32dll, 我也不知道为什么)

建立工程
打开vs2005,
在additionalinclude directories加上
“C:\openssl-0.9.8g\include”
这个目录.

在additionallibrary directories加上
“C:\openssl-0.9.8g\out32dll”
这个目录

代码类似下面这样:

编译不会有问题.同样在VC6下，把上面两个目录包含进来，编译也通过.

openssl 生成公钥 私钥命令行

Windows用户在cmd窗口中进行以下操作：
C:\Users\Hammer>cd C:\OpenSSL-Win32\bin 进入OpenSSL安装目录
C:\OpenSSL-Win32\bin>openssl.exe 进入OpenSSL程序
OpenSSL> genrsa -out rsa_private_key.pem 1024 生成私钥
OpenSSL> pkcs8 -topk8 -inform PEM -in rsa_private_key.pem -outform PEM -nocrypt Java开发者需要将私钥转换成PKCS8格式
OpenSSL> rsa -in rsa_private_key.pem -pubout -out rsa_public_key.pem 生成公钥
OpenSSL> exit ## 退出OpenSSL程序

证书文件的使用 .p12文件解析

同一个.p12证书怎么android获取到私钥是16进制的，java获取到的私钥是10进制。
android代码：
KeyStore ks = KeyStore.getInstance(“PKCS12”);
InputStream fis = ctx.getAssets().open(“MobileClient.p12”);
char[] pwd = “123456”.toCharArray();
ks.load(fis, pwd);
fis.close();
Enumeration enume = ks.aliases();
String keyAlias = null;
if (enume.hasMoreElements()) {
keyAlias = (String) enume.nextElement();
}
PrivateKey pk = (PrivateKey) ks.getKey(keyAlias, pwd);
System.out.println(pk);

java代码：
KeyStore ks = KeyStore.getInstance(“PKCS12”);
FileInputStream fis = new FileInputStream(“h:/MobileClient.p12”);
char[] pwd = “123456”.toCharArray();
ks.load(fis, pwd);
fis.close();
Enumeration enume = ks.aliases();
String keyAlias = null;
if (enume.hasMoreElements()) {
keyAlias = (String) enume.nextElement();
}
PrivateKey pk = (PrivateKey) ks.getKey(keyAlias, pwd);
System.out.println(pk);
